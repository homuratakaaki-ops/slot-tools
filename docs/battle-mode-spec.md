## 0. 目的と位置づけ

nerai-record.html に、実戦中専用の入力ビュー「実戦モード」を追加する。
既存の稼働入力画面は廃止せず、「管理・編集画面」として残す。記録の入口を実戦モードへ、詳細補完・修正の出口を既存画面へ、と役割分担する。

本仕様は**骨格のみ**を対象とする。逐次チップ・割り込み復帰・進行シーケンス・自動推測・並べ替え編集UIなどの拡張は対象外（§9参照）。

### ゴール指標（実装後の受け入れ基準に使用）

| 指標 | 目標 |
|---|---|
| 通常イベント記録 | 1タップで完了 |
| 誤入力からの復帰 | 1タップ |
| 実戦モード内のスクロール | 原則0回（縦530px級の画面で全要素が収まる） |
| 保存確認 | タップ直後にトースト＋振動で完結（確定ボタンなし） |
| G数表示と操作ボタン | 常に同一ビューポート内 |

---

## 1. スコープ

### やること（骨格5要素）

1. **画面分離**: 実戦モード専用ビューの新設。管理系UI（機種・店舗・項目・示唆マスタ・テンプレ）は実戦モードに一切表示しない
2. **固定グリッド**: クイックボタンを固定位置グリッドで配置。状態によって位置を動かさない（無効時はグレーアウトのみ）
3. **タップ即保存**: 確定操作なしで localStorage へ即書き込み
4. **常時取り消し**: 直前イベントの取り消しバーを画面下部に常駐。取り消し後の復元（Undo の Undo）にも対応
5. **同一画面化**: G数表示（データG／液晶G）とクイックボタンを同時に視認できるレイアウト

### やらないこと

- 既存の稼働入力画面・ログ画面・管理画面の改修（導線追加を除く）
- クイックボタンの並べ替え・表示切替のユーザー編集UI（データ構造のみ将来対応可能に設計）
- 新しいログ種別・データ層の追加（既存4層構造をそのまま使う）
- Wake Lock、長押し詳細、最近使用履歴などの拡張機能

## 2. 画面構成

### 2.1 モード切替

- 稼働入力タブ内、アクティブな稼働セッションが存在するときに「実戦モードで記録」ボタンを表示
- 実戦モードは全画面オーバーレイ（既存タブUIを覆う）として実装。SPA的なページ遷移は不要
- 実戦モード右上に「閉じる」→ 既存の稼働入力画面へ戻る。**閉じてもセッション状態・入力内容は一切変化しない**（表示の切替のみ）

### 2.2 実戦モード レイアウト（上から順に）

```text
┌──────────────────────────┐
│ 機種名 / 狙い方名        ✕ │  ← ヘッダ（1行、タップ不可）
├──────────────────────────┤
│  データG        液晶G      │  ← 現在値表示（数値20〜28px）
│    184           176       │     タップでテンキー（既存モーダル流用）
├────────────┬─────────────┤
│   ＋1G      │   ＋10G      │  ← 固定行（全機種共通）
├────────────┼─────────────┤
│  ボタン3    │  ボタン4     │
├────────────┼─────────────┤
│  ボタン5    │  ボタン6     │  ← 機種依存行（クイック定義から生成）
├────────────┼─────────────┤
│  ボタン7    │  その他 ▼    │  ← 「その他」は固定（右下）
├──────────────────────────┤
│ 直近: 184G スイカ 14:32     │  ← 直近1〜3件（読み取り専用）
├──────────────────────────┤
│ ↶ 直前「184G スイカ」を取消 │  ← 取り消しバー（常駐・画面最下部）
└──────────────────────────┘
```

### 2.3 寸法基準

| 項目 | 値 |
|---|---|
| クイックボタン高さ | 52〜64px |
| ボタン間隔 | 6〜10px |
| ボタン文字 | 16px以上 |
| G数数値 | 20〜28px |
| タップ領域 | 最低48×48px（見た目より広くてよい） |
| グリッド | 2列 × 4行（＋1G/＋10G行含む）を上限 |

縦に収まらない端末（表示領域が極端に低い場合）のみ、直近表示を1件に減らして吸収する。クイックボタンのサイズは削らない。

## 3. クイックボタン仕様

### 3.1 構成

- グリッドは最大8枠。うち3枠は固定: **＋1G／＋10G（上段）、その他（右下）**
- 残り最大5枠は機種プリセットの「クイック定義」から生成する

### 3.2 クイック定義（データ構造）

機種プリセットに以下を追加する。**編集UIは作らない**（v1ではプリセット定義内に静的に持つ）。

```js
quickPanel: [
  { type: "tag",   ref: "<既存タグID>" },          // 時系列タグを1タップ記録
  { type: "modal", ref: "suggestion" },            // 既存の示唆選択モーダルを開く
  { type: "modal", ref: "destiny" },               // 運命盤モーダル（対応機種のみ）
  // 最大5件
]
```

- `type: "tag"` … タップで該当タグのログを即時記録（§4）
- `type: "modal"` … 既存モーダルを開くだけ。モーダル側の挙動は一切変更しない
- クイック定義が未設定の機種では、機種依存枠を空欄（グレー枠）表示とし、実戦モード自体は使用可能とする

### 3.3 「その他」ボタン

- タップでボトムシートを開き、その機種の**全タグ**（低頻度タグ含む）を一覧表示
- 一覧からのタップも即時記録（クイックボタンと同じ経路）
- 既存の折りたたみ表示ロジックがあれば流用してよいが、実戦モード内では検索・管理機能は出さない

---

## 4. 記録フロー（タップ即保存）

### 4.1 基本フロー

1. ボタンタップ
2. 既存のログ追加処理（instantLog 系）を呼び、現在のセッション・区切りに紐づくログとして**即時に localStorage へ保存**
3. フィードバック（§5）を返す
4. 直近表示と取り消しバーを更新

確定ボタン・確認ダイアログは一切挟まない。

### 4.2 記録内容

- タップ時点で保存するのは **時刻・現在G数・タグ種別** のみ
- 数値やメモが必要な項目は保存後に既存の編集経路で補完する（本仕様では補完導線の新設はしない。既存ログ画面での編集に委ねる）

### 4.3 G数加算（＋1G／＋10G）

- タップで現在G数に加算し、即保存
- これはログイベントではなく現在値の更新として扱う（既存のG数管理の仕組みに従う）
- 取り消しバーの対象には**含める**（＋10Gの誤タップ復帰は頻出のため）

### 4.4 既存ロジックとの関係

- ログの生成・保存・区切りへの紐づけは**既存関数を呼び出す**こと。実戦モード側でログ構造を再定義しない
- スイカオフセット2層管理・セグメントライフサイクルに関わる処理は既存経路をそのまま通す。実戦モード起因で別経路を作らない

## 5. フィードバック仕様

| イベント | 振動 | 表示 |
|---|---|---|
| 記録成功 | 短1回（10〜30ms） | ボタン押下アニメ＋トースト「◯◯を記録」0.8秒 |
| 取り消し | やや長め1回 | トースト「◯◯を取り消し［元に戻す］」 |
| 無効操作 | 短2回 | トースト（理由を1行） |

- 振動は `navigator.vibrate` を使用。非対応端末では表示のみで成立させる
- 音は使用しない
- トーストは画面下部の取り消しバーの直上に表示し、ボタン群を隠さない

---

## 6. 取り消し仕様

- 取り消しバーは常駐。直前イベントの内容（例: `184G スイカ`）を表示
- タップで直前イベントを削除。**確認ダイアログは出さない**
- 削除後、トースト内の［元に戻す］で復元可能（トースト消滅までの間、最低5秒）
- 取り消し対象はスタックで管理し、連打で複数件さかのぼれる（上限は直近の区切り内まで）
- G数加算の取り消しは加算前の値へ戻す
- 区切りをまたぐ削除・確定済み収支に関わる削除は対象外とし、無効操作フィードバックを返す（既存画面で編集させる）

## 7. 既存コードへの影響範囲（ミコトへの制約）

**変更してよい範囲**

- 実戦モード用の新規ビュー・新規JSモジュール（関数・CSSは接頭辞 `bm` などで名前空間を分離）
- 稼働入力タブへの「実戦モードで記録」ボタン追加
- 機種プリセットへの `quickPanel` フィールド追加（後方互換: 未定義でも既存動作に影響しないこと）

**変更してはいけない範囲**

- 既存のログデータ構造・localStorage スキーマ（フィールド追加が必要な場合は事前に相談）
- 既存モーダル（テンキー・示唆選択・運命盤・獲得枚数）の内部挙動。呼び出すだけにする
- スイカオフセット・セグメントライフサイクル・エンディング集計・PNG共有カードの各ロジック
- 既存の稼働入力画面のDOM構造（ボタン追加箇所を除く)

機種プリセットに触れるため、docs/machine-preset-guide.md の参照と**3系統検証**を必ず実施すること。

---

## 8. 検証項目

1. 実戦モードで記録したログが、既存のログ画面・整理ビュー・PNG共有カード・エンディング集計に既存記録と同一形式で反映される
2. 実戦モード⇔既存画面の往復でセッション状態が保持される（リロード含む）
3. 取り消し→元に戻す→再取り消しでデータ不整合が出ない（スイカ履歴・オフセット含む）
4. quickPanel 未定義の機種で実戦モードが劣化なく動作する
5. 3系統検証（機種プリセット変更ルールに基づく既存3機種系統での回帰確認）
6. オフライン状態での記録・取り消しが成立する
7. AQUOS Sense9 実機で §0 のゴール指標を満たす（スクロール0回・1タップ記録）

---

## 9. 非スコープ（IDEAS控え）

以下は骨格検証後の拡張候補。本改修には含めない。

- 逐次チップ（当選直後に種別チップを数秒表示）
- 割り込み復帰（離脱→復帰時のG数更新プロンプト）
- クイックボタンの並べ替え・編集UI
- 確認省略/即時記録のモード切替設定
- 進行シーケンス（区切りチェーンの事前定義進行）
- 長押し詳細入力・最近使用履歴・Wake Lock・自動推測
