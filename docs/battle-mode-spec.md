## 0. 目的と位置づけ

nerai-record.html に、実戦中専用の入力ビュー「実戦モード」を追加する。
既存の稼働入力画面は廃止せず、「管理・編集画面」として残す。記録の入口を実戦モードへ、詳細補完・修正の出口を既存画面へ、と役割分担する。

v1.1 の設計方針: 現行の実戦メモ画面（カウンター操作〜レア役・ステージ・メモ）は、適切な表示位置を保てばスクロールなしで運用できることが実戦で確認されている。したがって実戦モードはこの**実績あるレイアウトを構造として固定**し、そこに取り消しバー・振動フィードバックを追加し、管理系UIを排除したものとして実装する。ゼロから新レイアウトを設計しない。

### ゴール指標（受け入れ基準）

| 指標 | 目標 |
|---|---|
| カウンター系イベント記録（リプレイ等） | 1タップで完了 |
| ピッカー系イベント記録（さざなみ・ボーナス） | 2〜3タップで記録開始まで完了 |
| 誤入力からの復帰 | 1タップ |
| 実戦モード内のスクロール | 原則0回 |
| 保存確認 | タップ直後にトースト＋振動で完結（確定ボタンなし） |
| G数表示と主要操作 | 常に同一ビューポート内 |

## 1. スコープ

### やること（骨格）

1. **画面分離**: 実戦モード専用ビューの新設。管理系UI（機種・店舗・項目・示唆マスタ・テンプレ）、および実戦中に不要な操作（「確定にする」等の集計確定系）は実戦モードに表示しない
2. **固定配置**: §2 のゾーン構成を固定。状態によってボタン位置を動かさない（無効時はグレーアウトのみ）
3. **タップ即保存**: 確定操作なしで localStorage へ即書き込み
4. **常時取り消し**: 直前イベントの取り消しバーを画面下部に常駐。取り消し後の復元にも対応
5. **統一ピッカー**: さざなみ・ボーナスの2つを統一ボタン→選択式で定義
6. **同一画面化**: G数表示と主要操作を同時に視認できるレイアウト

### やらないこと

- 既存の稼働入力画面・ログ画面・管理画面の改修（導線追加を除く）
- クイックボタンの並べ替え・表示切替のユーザー編集UI
- 新しいログ種別・データ層の追加（既存4層構造・timeline形式をそのまま使う）
- 逐次チップ・割り込み復帰・進行シーケンス・自動推測・Wake Lock 等の拡張（§9）

## 2. 画面構成

### 2.1 モード切替

- 稼働入力タブ内、アクティブな稼働セッションが存在するときに「実戦モードで記録」ボタンを表示
- 実戦モードは全画面オーバーレイとして実装
- 右上「✕」で既存画面へ戻る。**閉じてもセッション状態・入力内容は一切変化しない**（表示の切替のみ）

### 2.2 ゾーン構成（上から順、現行実戦メモ画面を踏襲）

```text
┌──────────────────────────────┐
│ 機種名 / 狙い方名           ✕ │ ← ヘッダ
├──────────────────────────────┤
│ データG数 [184]  液晶G数 [176] │ ← G数表示（タップで既存テンキー）
│ 差分 [0] [+9]   スイカ [0] [+1]│ ← 現行のカウンター操作を踏襲
├──────────────────────────────┤
│ [＋1]    [＋5]    [＋10]      │ ← G加算段（現行踏襲）
├──────────────────────────────┤
│ [リプレイ] [チェリー] [スイカ] │ ← カウンター段（現行踏襲・1タップ）
├──────────────────────────────┤
│ [プリリプ] [さざなみ] [ボーナス]│ ← イベント段（プリリプ=tag、
│                    [その他 ▼] │    さざなみ/ボーナス=ピッカー）
├──────────────────────────────┤
│ 直近: 184G さざなみ赤 14:32    │ ← 直近1〜3件（読み取り専用）
├──────────────────────────────┤
│ ↶ 直前「184G スイカ」を取消    │ ← 取り消しバー（常駐・最下部）
└──────────────────────────────┘
```

- 現行実戦メモ画面との差分は、**追加**: 取り消しバー・直近表示・振動フィードバック、**削除**: 「確定にする」「示唆記録」大ボタン等の管理・確定系、さざなみ紫/赤/虹の個別3ボタン、リプレイフラッシュ個別ボタン（→その他へ）、**統合**: BIG/REG→ボーナスピッカー、さざなみ3色→さざなみピッカー
- 示唆記録は「その他 ▼」内から既存モーダルを呼べる形で残す（実戦中の使用頻度が低いため一等地から外す）

### 2.3 寸法基準（v1.0 から変更なし）

| 項目 | 値 |
|---|---|
| ボタン高さ | 52〜64px |
| ボタン間隔 | 6〜10px |
| ボタン文字 | 16px以上 |
| G数数値 | 20〜28px |
| タップ領域 | 最低48×48px |

縦に収まらない端末のみ、直近表示を1件に減らして吸収する。ボタンサイズは削らない。

## 3. ボタン仕様

### 3.1 ボタン型の種類

| type | 挙動 |
|---|---|
| counter | タップで既存カウンターを+1し即保存（リプレイ／チェリー／スイカ、G加算） |
| tag | タップで該当タグを timeline へ即時記録（プリリプ等） |
| picker | タップで選択シートを開き、選択完了で記録（さざなみ／ボーナス） |
| modal | 既存モーダルを開くだけ（その他内の示唆記録等） |
| hit | タップで既存の当選記録ウィザードを起動（§3.5） |

### 3.2 「その他 ▼」

- タップでボトムシートを開き、その機種の全タグ（低頻度タグ・リプレイフラッシュ含む）と示唆記録モーダルへの導線を表示
- 一覧からのタグタップも即時記録

### 3.3 さざなみピッカー

- 「さざなみ」タップでボトムシートを開く
- **ステップ1（契機・6択）**: プリリプ／スイカ3回目／スイカ5回目／スイカ7回目／スイカ9回目／それ以外
- **ステップ2（色・3択）**: 紫／赤／虹
- 色の選択と同時に記録・保存し、シートを閉じる（確定ボタンなし）
- 当選有無はここでは扱わない（当選は別経路＝ボーナスピッカー/hitTriggers の責務）
- **保存形式**: 既存の さざなみ紫/赤/虹 の3タグのいずれかとして timeline へ保存し、契機を entryCause / entryCauseLabel に格納する（§4.5）。新タグは作らない
- シート表示中に画面外タップまたは「閉じる」で、**何も記録せず**キャンセル

### 3.4 ボーナスピッカー

- 「ボーナス」タップでボトムシートを開く
- **1ステップ・3択**: 赤BIG／青BIG／REG
- 選択と同時に、対応する当選記録ウィザードを起動（§3.5）
- **対応表（L南国）**:

| 選択肢 | 内部 trigger | variant | 現行表示との関係 |
|---|---|---|---|
| 赤BIG | direct_at | なし（現行デフォルトのまま） | 現行「BIG当選」 |
| 青BIG | direct_at | blue7 | 現行「BIG当選」＋青7チップ |
| REG | episode_bonus | なし | 現行「REG当選」 |

- 赤BIGに新variantを追加しない。過去ログ（variant無しのBIG）との連続性を保つため、variant無し＝赤BIGの意味とする
- STEP3側の青7チップUIは現状維持（実戦モード外からの記録・修正用）。ピッカーからの起動時は該当variantが選択済み状態でウィザードに入ること

### 3.5 hit型の挙動

- タップ（ピッカー選択）で `startHitEventWizard('<trigger>')` 相当の**既存の当選記録フロー**を起動する
- 起動後は既存フロー（AT当選ルート→獲得枚数→終了時示唆等）にそのまま従う。**新しい入力経路・省略経路を作らない**
- 設計意図: カウンター系は遊技中の高頻度操作のため1タップ、当選系はボーナス消化中で時間圧が低いため既存の流れで示唆まで完結させる。獲得枚数・終了時示唆を後回しにすると記録漏れが起きるため、当選時はあえて多段入力を許容する
- ウィザード起動中は実戦モードのボタン群を覆ってよい（ウィザード完了/中断後に実戦モードへ戻る）
- ウィザードを中断（ルート解除等）した場合の挙動は既存仕様に従う

### 3.6 L南国育ちSPECIAL quickPanel（確定版）

```js
quickPanel: {
  counters: ["replay", "cherry", "suika"],        // 既存カウンターID へ解決
  gainButtons: [1, 5, 10],
  events: [
    { type: "tag",    ref: "<プリリプのタグID>" },
    { type: "picker", ref: "sazanami" },
    { type: "picker", ref: "bonus" }
  ]
}
```

- ref は実装時に既存タグマスタ・カウンター定義の実IDへ解決すること
- 該当タグ・カウンターが未登録の場合は実装前に報告し、勝手に新設しない
- quickPanel 未定義の機種では、カウンター段・G加算段・「その他 ▼」のみで実戦モードを成立させる（イベント段は空欄表示）

## 4. 記録フロー（タップ即保存）

### 4.1 基本フロー

1. ボタンタップ（picker は選択完了時点）
2. 既存のログ追加処理を呼び、現在のセッション・区切りに紐づけて**即時に localStorage へ保存**
3. フィードバック（§5）
4. 直近表示と取り消しバーを更新

確定ボタン・確認ダイアログは挟まない（hit型のウィザード内は既存仕様に従う）。

### 4.2 記録内容

- counter/tag/picker のタップ時点で保存するのは **時刻・現在G数・タグ種別（＋picker の選択内容）**
- 数値やメモの補完は既存ログ画面での編集に委ねる

### 4.3 G数加算（＋1／＋5／＋10）

- タップで現在G数に加算し、即保存。既存のG数管理の仕組みに従う
- 取り消しバーの対象に含める

### 4.4 既存ロジックとの関係

- ログの生成・保存・区切りへの紐づけは既存関数を呼ぶ。実戦モード側でログ構造を再定義しない
- スイカオフセット2層管理・セグメントライフサイクルに関わる処理は既存経路をそのまま通す

### 4.5 さざなみ契機の付帯情報（entryCause の意味拡張）

- さざなみタグの timeline entry に、契機を entryCause / entryCauseLabel で格納する
- **意味拡張の明記**: entryCause は従来「SDC/超飛翔突入の契機」の用途で使われている。本仕様により「timelineタグの契機付帯」へ意味を拡張する
- **値の列挙（固定）**: さざなみ用の entryCause は以下のみとする

| entryCause | entryCauseLabel |
|---|---|
| sazanami_pururipu | プリリプ |
| sazanami_suika3 | スイカ3回目 |
| sazanami_suika5 | スイカ5回目 |
| sazanami_suika7 | スイカ7回目 |
| sazanami_suika9 | スイカ9回目 |
| sazanami_other | それ以外 |

- 値には接頭辞 `sazanami_` を必須とし、既存のSDC/超飛翔系の entryCause 値と名前空間を分離する。**既存値と衝突しないことを実装時に確認し、既存値の一覧を報告すること**（進行中の有利区間差枚・SDCトリガー分析が entryCause を参照している可能性があるため）
- attachments は value 数値必須のため本用途には使用しない

## 5. フィードバック仕様（v1.0 から変更なし）

| イベント | 振動 | 表示 |
|---|---|---|
| 記録成功 | 短1回（10〜30ms） | ボタン押下アニメ＋トースト「◯◯を記録」0.8秒 |
| 取り消し | やや長め1回 | トースト「◯◯を取り消し［元に戻す］」 |
| 無効操作 | 短2回 | トースト（理由を1行） |

- `navigator.vibrate` を使用。非対応端末では表示のみで成立。音は使用しない
- トーストは取り消しバーの直上に表示し、ボタン群を隠さない

## 6. 取り消し仕様（v1.0 から変更なし、対象を追記）

- 取り消しバーは常駐。直前イベントの内容を表示
- タップで直前イベントを削除。確認ダイアログは出さない
- 削除後、トースト内の［元に戻す］で復元可能（最低5秒）
- スタック管理で連打により複数件さかのぼれる（上限は直近の区切り内まで）
- **対象**: counter（リプレイ等・G加算含む）、tag、picker で記録したイベント
- **対象外**: hit型ウィザードで確定した当選記録（既存の編集経路で修正させる）、区切りをまたぐ削除、確定済み収支。対象外の場合は無効操作フィードバックを返す
- G数加算・カウンターの取り消しは加算前の値へ戻す（スイカ履歴・オフセットとの整合を含む）

## 7. 既存コードへの影響範囲（ミコトへの制約）

**変更してよい範囲**

- 実戦モード用の新規ビュー・新規JSモジュール（接頭辞 `bm` 等で名前空間分離）
- 稼働入力タブへの「実戦モードで記録」ボタン追加
- 機種プリセットへの quickPanel フィールド追加（後方互換: 未定義でも既存動作に影響しないこと）
- さざなみタグ保存時の entryCause / entryCauseLabel 付与（§4.5 の列挙値のみ）

**変更してはいけない範囲**

- 既存のログデータ構造・localStorage スキーマ（§4.5 の範囲を除く。追加フィールドが必要な場合は事前相談）
- 既存モーダル・当選記録ウィザードの内部挙動（呼び出しと variant 初期選択の受け渡しのみ可）
- スイカオフセット・セグメントライフサイクル・エンディング集計・PNG共有カードの各ロジック
- 既存の稼働入力画面のDOM構造（ボタン追加箇所を除く）

機種プリセットに触れるため、docs/machine-preset-guide.md の参照と**3系統検証**を必ず実施すること。

## 8. 検証項目

1. 実戦モードで記録したログ（counter/tag/picker）が、既存のログ画面・整理ビュー・PNG共有カード・エンディング集計に既存記録と同一形式で反映される
2. ボーナスピッカー経由の当選記録が、既存のBIG/REG＋青7チップ経由の記録と同一形式になる（3択それぞれで確認）
3. さざなみピッカーの記録が既存3タグとして保存され、entryCause が §4.5 の列挙値で格納される。既存のSDC/超飛翔系 entryCause を参照する処理（分析・集計・表示）に影響がない
4. 実戦モード⇔既存画面の往復でセッション状態が保持される（リロード含む）
5. 取り消し→元に戻す→再取り消しでデータ不整合が出ない（スイカ履歴・オフセット・カウンター値含む）
6. quickPanel 未定義の機種で実戦モードが劣化なく動作する
7. 3系統検証（機種プリセット変更ルールに基づく回帰確認）
8. オフライン状態での記録・取り消しが成立する
9. AQUOS Sense9 実機で §0 のゴール指標を満たす（夢爽側で実施）

## 9. 非スコープ（IDEAS控え）

- 逐次チップ（記録直後の詳細チップ表示）
- 割り込み復帰（離脱→復帰時のG数更新プロンプト）
- クイックボタンの並べ替え・編集UI
- 確認省略/即時記録のモード切替設定
- 進行シーケンス（区切りチェーンの事前定義進行）
- 長押し詳細入力・最近使用履歴・Wake Lock・自動推測
- リプレイ等カウンター実測値を使った分母分析の強化（将来のIDEAS候補）
