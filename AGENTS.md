# AGENTS.md

## 機種プリセット作業ルール

`nerai-record.html` の機種プリセット追加・変更を行う場合は、作業前に必ず `docs/machine-preset-guide.md` を読むこと。

`nerai-record` は実戦記録に徹する。狙い目判断・検証・分析の機能は持たせず、記録データを別ツールで活用する前提とする。

検証は同ガイドの3系統を最低ラインとする。

- 新規環境
- 既存データ再現
- 対象外機種

作業中に新しい落とし穴、汎用機構、検証観点を発見した場合は、`docs/machine-preset-guide.md` への追記を提案し、必要なら別コミットで反映すること。

機種プリセット以外の作業では、このルールを無理に適用しない。

## nerai-record データ保護ルール

`nerai-record.html` の起動時処理、特に `loadData()` と正規化処理は、後方定義の `const` / `let` / 関数初期化順に依存させないこと。保存済み旧形式データを読み込む前に、参照される定数・補正テーブル・正規化関数がすべて初期化済みである状態を保つ。

旧形式データのロード経路を変更した場合は、`tests/nerai-record-regression.test.js` を実行し、少なくとも以下を確認すること。

- 旧 `stores` 文字列配列が現行の店舗オブジェクト形式へ正規化される
- 旧東京喰種データのロードで例外・空フォールバックが発生しない
- 旧データに合成ログを加えたケースで、空化保存ガードが保存拒否として働く
